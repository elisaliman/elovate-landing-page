# Outfit asset inspection — current state

Generated by `uv --project tools/normalize_outfits run normalize-outfits inspect`.

## File-format note

Despite the `.jpg` extension on outfit_2 and outfit_3, every file is actually a
PNG with a real alpha channel (verified via PIL `format=PNG, mode=RGBA`). The
tool keys off magic bytes, not extension, so background detection is never
needed for the current asset set. If you add real JPGs later, the fallback
perimeter-sampling background mask in `imageio.load_rgba` will kick in.

Highlights:

- **Outfit 1's shirt** is cropped edge-to-edge — bbox equals canvas, no padding.
- **Outfit 1's belt and shoes** are anchored low on their canvases (cy = 0.65
  and 0.73). This isn't a deliberate composition choice; it's how the source
  cutouts happened to be framed. Our config centralizes anchors so swapped
  pieces don't drift visually.
- **Outfit 3's belt** is tiny (410×410). It just gets upscaled by the pipeline.
- **Pants** are the most aspect-consistent across the set (σ=0.05 on bbox aspect).

All assets in a category now share the same canvas dimensions, the same
vertical anchor, and the same horizontal centerline. What variation remains is
the natural difference in bbox aspect ratio between specific garments — which
we *don't* want to flatten, since deforming images would look worse than the
small leftover variation.
